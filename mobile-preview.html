makeLILA — Mobile View Preview v2 (card-drilldown pattern)

Vertical scrollable list of clickable module cards on the home screen. Tap a module to drill in — that screen shows tab cards (also clickable). Tap a tab to see its content. Each level uses the iOS-native "back ‹" pattern. Three levels deep: Home → Module → Tab → Content.

The new mobile navigation model

  1. Home — instead of a desktop top-nav strip, the home screen is a vertical scrollable list of module cards. Each card shows the module icon, name, a one-line description, and a count pill on the right (number of items that need attention). Tap → drill into module.
  2. Module screen — instead of a horizontal tab strip, tabs become tab cards stacked vertically. Same card design as home. Tap → drill into tab.
  3. Tab screen — the actual records (orders, tickets, customers) as content cards. Tap → opens the detail takeover.
  4. Detail takeover — full-screen, sticky header (back + title), sticky bottom action bar (Approve / Reject / etc.).
  5. Navigation primitives: ‹ back button top-left on every non-home screen wires to the browser back stack, so iOS swipe-back works natively. Right-side header slot reserved for context actions (Filter / Add / etc.).

Why this works better on phone

  1. Every card is >60px tall — Apple HIG-compliant tap target with room to spare.
  2. Count pills surface "what needs me right now" — operator doesn't have to drill in just to check.
  3. Two visual groups ("Today's attention" / "Workspace") let high-priority modules float to the top of the list.
  4. Module description line gives newcomers (e.g., Julie on her first phone session) context that the desktop nav assumes you've memorised.
  5. On desktop the same React component renders the existing horizontal nav — the card-drilldown only kicks in under @media (max-width: 700px).

How the drill works for OrderReview

  1. From Home, tap "Order Review" card → land on Module screen (left frame).
  2. Tap a tab card (e.g., "Pending") → land on Tab screen (right frame) showing the 7 pending orders as content rows.
  3. Tap a row → opens Detail takeover (see "Detail panel" chip above).
  4. Each tab card surfaces its own count + a description, so the operator knows what's behind the door without tapping.
  5. Search bar appears on the Tab screen for tabs with >15 rows.
