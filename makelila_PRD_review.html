Risks & Open Questions

Items where the gap analysis surfaced decisions or risks the team should resolve before committing to the timeline.

Open product decisions

  • HubSpot decommission timing. Depends on Klaviyo Track API firehose being live first (P1). Pedrum sign-off required on sequence-migration feasibility.
  • Klaviyo SMS adoption. Klaviyo for sequence-driven SMS vs Quo as sole SMS surface. Recommendation: split (Klaviyo for sequence, Quo for 1-on-1).
  • Pre-sale ad-spend scope. Facebook only for V1, or include Google Ads + TikTok? Affects §5 Campaign dashboard data model.
  • Customer-facing portal. Deferred to P3 — revisit when "where is my order" volume exceeds 20/week.
  • Lead-form ownership post-HubSpot. Recommendation: lila.vip forms post to makeLILA endpoint, with optional Klaviyo profile creation downstream.
  • Sezzle direct integration. P3 candidate — inferred via Shopify today, may be worth direct API for QBO reconciliation.

Timeline risks

  • QBO API multi-currency. CAD/USD ledger discipline is a known gotcha. Add 1 week of validation cycle for refund + replacement edge cases.
  • Facebook Marketing API auth. Meta Business System User access tokens have a separate approval flow. Add 1–2 weeks lead time before P1 work can start.
  • Telemetry-driven ticket auto-creation false-positive rate. Backlog #70 documented 75% false positive on NOT_MIXING. Need to validate auto-create thresholds with a 2-week shadow run before flipping the switch.
  • RBAC migration on live DB. Promoting from email-list checks to profiles.role with RLS tightening is single-shot. Schedule the migration window with operator notice.
  • Mobile V1 PWA install testing on iOS. Safari PWA behaviors require physical-device testing; allow 1 extra validation week.
  • Sales projection model accuracy. Rolling-average baseline has no validation cohort until ~30 days of data accumulates. Pipeline-driven variant defers to P3 specifically to give the rolling-average a chance to calibrate first.

Recommended sequencing summary

  1. Week 1–2: RBAC profiles + canDo/canView + Klaviyo Track API firehose + PaymentCard UI (S items, parallel).
  2. Week 3–6: Returns/Refunds overhaul finishing (grade + disposition + quarantine + #79 net out) — closes the biggest active operator pain.
  3. Week 4–8: Device-context header + warranty registration + SLA aging + telemetry-driven ticket auto-create — Service-domain hardening, runs parallel to Returns finish.
  4. Week 6–10: Finance module skeleton + JournalPanel ships behind RBAC. Depends on Returns being clean.
  5. Week 8–12: Facebook Marketing API + Lead Ads webhook + mobile V1. Pre-sale visibility lands.
  6. Week 12–18: P2 batch — build_station_passes, Klaviyo predictive pull, freight quote history, activity_log entity refs, ProductionProjectionPanel, OKR/KPI tracking, CAPI, mobile V2, QBO full daily-summary, HubSpot decommission window.
  7. Week 18–24: Stabilization + customer-facing roll-out + P3 revisit gate.
